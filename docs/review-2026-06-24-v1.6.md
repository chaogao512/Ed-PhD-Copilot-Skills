# Review: Ed-PhD-Copilot-Skills V1.6

## Review Scope

This review covers the V1.6 pressure-test and disclosure-template update. It evaluates whether the skill set has moved beyond static reference completeness toward realistic doctoral-topic execution, renderable figure validation and venue-aware AI disclosure.

## Overall Judgment

`Ed-PhD-Copilot-Skills` now reaches **V1.6 pressure-test enhanced status**. The strongest improvement is that the repository no longer only claims that a seven-skill chain exists; it now includes a realistic doctoral-topic pressure test that checks whether the core chain preserves the same governance object across idea evaluation, paper design, evidence planning and pre-submission review.

V1.6 also closes two V1.5 gaps:

- figure guidance now has a renderable Mermaid artifact plus a QA report;
- AI disclosure is no longer only generic, because venue-specific templates now cover Chinese journals, doctoral defense, international journal submission and ethics review.

## What Improved

### 1. A real doctoral-topic chain is now visible

The new pressure test uses a county education data governance topic, which is closer to the user's professional direction than the earlier smart-campus example alone. It tests whether the skills can avoid three common failures: platform-construction drift, unsupported effectiveness claims and hidden data-governance risks.

### 2. Figure design now has an artifact-level QA surface

The Mermaid diagram and QA report make figure quality inspectable. The diagram shows data subjects, data sources, governance capacity, educational use, oversight and a dataset datasheet panel. This gives future revisions a concrete object to improve rather than only a prose instruction.

### 3. AI disclosure is venue-aware

The new templates distinguish Chinese CSSCI/core journal submission, doctoral proposal/dissertation defense, international journal submission and ethics review. This matters because a single generic AI disclosure sentence is often too weak for doctoral and journal contexts.

### 4. Publisher-policy coverage is broader

Wiley guidance was added as an additional publishing-ethics anchor alongside ICMJE, Nature and Elsevier. The skill correctly treats these sources as disclosure/accountability anchors, not as research-method evidence.

### 5. The skill chain better protects claim boundaries

The pressure test explicitly downgrades claims from "digitalization improves education quality" to more supportable mechanism claims unless stronger longitudinal or quasi-experimental evidence exists.

## Remaining Gaps

### 1. Real installed-skill trigger testing is still pending

The repository has strong internal validation, but the skills have not been installed into the active Codex skill environment in this workflow. Installation remains intentionally deferred because the user asked to改造后再安装.

### 2. Chinese CSSCI/core article verification remains the main academic gap

Chinese official policy sources are verified, but peer-reviewed Chinese journal articles still require CNKI, journal-site, DOI, local PDF or school database metadata before they can become formal citation anchors.

### 3. Only one figure has artifact-level QA

The education data governance figure is now renderable. Teacher digital competence and human-AI assessment figures still have text-level outputs only.

### 4. Venue templates still need target-specific final checks

The templates are useful defaults, but final wording must still be checked against the named journal, university or ethics board when a real submission venue is known.

### 5. Pressure testing covers the core chain, not all seven skills

The V1.6 pressure test covers the four most critical skills. A V1.7 or V2.0 release should run the full seven-skill chain on one authentic doctoral topic.

## Next Improvement Priorities

### P1: Full seven-skill pressure test

Run the county education data governance topic through all seven skills and record where each skill changes or preserves the research object.

### P2: Chinese article verification

Use CNKI, journal websites, DOI pages, local PDFs or school database exports to verify a small number of high-confidence Chinese journal articles. Promote only verified entries into `docs/chinese-core-literature-verified.md`.

### P3: More rendered figures

Create Mermaid or SVG/PPT-ready artifacts for:

- teacher digital competence governance;
- human-AI classroom assessment;
- doctoral proposal research design flow.

### P4: Installed-skill forward test after user approval

After installation is explicitly approved, test the seven trigger prompts in the active Codex environment and record whether each skill loads the correct references.

### P5: Dissertation-stage templates

Add short templates for 开题报告, 中期考核, 预答辩 and 正式答辩 outputs, especially for research design, ethics disclosure and contribution statements.

## Verdict

Mark the repository as **V1.6 pressure-test enhanced**. It is now strong enough to serve as a serious doctoral research copilot template before installation. It is still not final because installed-skill testing and verified Chinese journal-article integration remain necessary for a fully mature V2.0 doctoral research workbench.
