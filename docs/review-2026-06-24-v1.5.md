# Review: Ed-PhD-Copilot-Skills V1.5

## Review Scope

This review covers the V1.5 method-transparency update: reporting-standard gates added to `mixed-methods-evidence-template`, method and ethics audits added to `edtech-pre-submission-reviewer`, source-registry updates, coverage-matrix updates and README maturity alignment.

## Overall Judgment

`Ed-PhD-Copilot-Skills` now reaches **V1.5 method-transparency enhanced status**. Compared with V1.4, the repository no longer only asks whether a study has mixed evidence; it now checks whether the evidence is reportable under recognizable methodological norms.

The largest improvement is that the skill set can more reliably prevent common doctoral-research overclaims:

- calling a convenience literature scan a systematic review;
- treating interview impressions as strong mechanism evidence without COREQ-style transparency;
- claiming causal effects from satisfaction surveys, one-group posttests or weak comparison designs;
- using model accuracy as proof of educational value or AI safety;
- describing high-risk education AI as governed without oversight, appeal, logging and monitoring.

## What Improved

### 1. Method claims are now bounded by reporting standards

The mixed-methods skill now tells the agent when to use PRISMA-style logic, COREQ-style qualitative reporting, CONSORT/SPIRIT and WWC-style intervention boundaries, and MMAT-style mixed-methods integration checks. This makes the skill more useful for doctoral proposals and CSSCI/SSCI-style review.

### 2. AI governance is treated as a lifecycle problem

The pre-submission reviewer now asks for context mapping, risk identification, risk measurement, risk management and monitoring. This is a major improvement over generic “privacy protection” paragraphs.

### 3. Claim downgrading is more precise

The skill now gives safer rewrites for overclaims, such as replacing “显著提升学习成效” with perceived usefulness or preliminary evidence when the data are weak.

### 4. Source traceability improved

The verified source registry now separates topical policy sources from method-reporting standards. This helps prevent a common problem: using a policy or ethics document as if it validated an empirical method.

### 5. Audit depth is now adjustable

The mixed-methods and pre-submission skills now support `Light / Standard / Strict` depth. This keeps the skill useful for early-stage doctoral brainstorming while preserving strict reporting checks for submission and defense contexts.

### 6. Chinese citation integrity is now explicit

A public-web verification log was added for Chinese EdTech literature candidates. The key gain is not a larger citation list, but a safer rule: public search snippets and AI-generated bibliographies cannot become formal Chinese core/CSSCI citations without CNKI, journal-site, DOI, local-PDF or school-database metadata.

### 7. AI publication-ethics guidance is now stronger

The AI-assisted workflow now includes authorship, disclosure, accountability and confidentiality rules aligned with major publication-policy anchors. It explicitly states that AI cannot be an author, humans remain responsible, material AI use should be recorded/disclosed according to venue rules, and sensitive education data or confidential manuscripts must not be uploaded to unapproved external AI tools.

### 8. Introduction and figure gates now cover high-risk AI representation

The introduction and figure skills now include V1.5 gates for evidence-synthesis wording, model/system-card panels, dataset datasheet panels, risk registers and responsibility swimlanes. This reduces the risk that a manuscript presents AI or data infrastructure as neutral technology while hiding intended use, non-use boundary, data rights, monitoring and accountability.

## Remaining Gaps

### 1. V1.5 rules are only partly propagated into examples

The mixed-methods and pre-submission examples now show PRISMA/COREQ/CONSORT/MMAT/NIST-style gates, but figure, introduction and AI-workflow examples could still demonstrate these standards more explicitly when they touch evidence synthesis or high-risk AI.

### 2. Chinese CSSCI/core-journal verification remains incomplete

Chinese official policy sources are verified, but Chinese journal article records still need CNKI, journal-site, DOI or local-PDF verification before being used as formal academic citations.

The 2026-06-24 public-web pass confirmed that open web search alone is not reliable enough for representative candidates such as 祝智庭、黄荣怀、顾小清/顾小青 and education data governance topic searches. This should be treated as a verification boundary, not as evidence that the literature does not exist.

### 3. Real installed-skill trigger testing is still pending

The repository has passed installation-forward simulation, but not real installed-skill testing in the active Codex skill environment.

### 4. Audit-depth modes need example validation

The `Light / Standard / Strict` modes have been added to the skill entry files, but example outputs should still demonstrate how the same topic is treated differently at each depth.

### 5. Venue-specific AI disclosure remains partly external

The skill now includes generic AI disclosure guidance, but actual disclosure wording still depends on the target journal, university and funder. Future use should add venue-specific checks when the user names a journal or degree-granting institution.

### 6. Real visual QA is still manual

The figure skill now specifies stronger visual content, but it has not yet been tested by rendering actual figures in PowerPoint/Figma/diagram tools. Future validation should include whether the recommended model-card and dataset-card panels remain readable at journal column size.

## Next Improvement Priorities

### P1: Extend V1.5 gates into the remaining examples

After the mixed-methods and pre-submission examples, update the remaining examples where relevant:

- introduction examples that use literature-review language;
- figure examples involving high-risk AI or data governance;
- AI-assisted workflow examples involving citation verification and source screening.

### P2: Add real doctoral-topic pressure test

Use one authentic doctoral topic from the user’s research direction and run the core chain:

`governance-idea-evaluator -> governance-paper-template -> mixed-methods-evidence-template -> edtech-pre-submission-reviewer`

The test should record where the V1.5 gates changed the output.

### P3: Continue Chinese literature verification

Move verified Chinese journal records from `docs/chinese-core-literature-inventory.md` into `docs/chinese-core-literature-verified.md` only when full bibliographic metadata and source access are confirmed.

Recommended next route:

1. Use CNKI or the user's institutional library access to export exact metadata.
2. Verify local PDFs when available.
3. Add a small number of high-confidence Chinese articles first, rather than many weak entries.
4. Convert each verified paper into one concrete skill rule or writing pattern.

### P4: Validate audit-mode selection

| Mode | Use when | Output depth |
|---|---|---|
| Light | Early idea exploration | 3-5 key risks |
| Standard | Proposal or manuscript planning | Full claim-evidence matrix |
| Strict | Pre-submission or defense preparation | Reporting-standard gate plus 72-hour repair plan |

### P5: Add venue-specific AI disclosure templates

When the target venue is known, add a short template for:

- Chinese CSSCI journal submission;
- doctoral dissertation or proposal defense;
- international journal submission;
- internal ethics review.

### P6: Render and audit one real figure

Use the human-AI assessment or education data governance case to produce an actual diagram, then check:

- whether actor lanes and responsibility flows are readable;
- whether model/system-card and dataset-card panels fit;
- whether high-risk AI boundaries are visible without overcrowding;
- whether the caption states the figure argument rather than describing boxes.

## Verdict

Mark the repository as **V1.5 method-transparency enhanced**. It is stronger and more scientifically disciplined than V1.4, especially for mixed-methods and AI governance studies. It is not yet “final” because example outputs and installed-skill testing still need to catch up with the new rules.
