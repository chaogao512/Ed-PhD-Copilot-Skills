## 仍需改进的问题

### 1. 中文题录需要精确核验

`chinese-core-literature-inventory.md` 已有 28 条，但其中一部分是 `needs_cnki_verification` 或 `local_plan_reference`。这些条目可用于技能模式设计，但不应作为正式论文引用。

**建议**：

- 使用 CNKI、期刊官网或本地文献库逐条补齐题名、作者、期刊、年份、卷期、页码、DOI。
- 新建 `docs/chinese-core-literature-verified.md` 保存已核验题录。

### 2. 需要更多真实论文 worked examples

当前样例以“多 Agent 智能校园治理”为单一贯穿案例。V1.0 后续可扩展：

- 教师数字素养与 AI 教师发展案例。
- 教育数据治理与隐私保护案例。
- 人机协同课堂质量评价案例。
- 区域教育数字化治理案例。

### 3. 自动化测试仍是结构级

脚本能检查文件存在、引用和基本厚度，但不能判断内容学术质量。

**建议**：

- 为 failure case 增加机器可读 metadata。
- 建立人工复评 checklist。
- 后续可引入样例输出对照审查。

## 是否可作为当前阶段完善版

可以。若“完善”定义为适合安装前的高质量技能库，V0.6 已达到可用标准。若“完善”定义为完全学术可引用、覆盖中文核心文献精确题录和多案例验证，则仍需 CNKI/本地文献核验。
