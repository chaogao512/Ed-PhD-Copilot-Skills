# Claim-Evidence Matrix

| Claim type | Minimum evidence | Insufficient evidence | Conclusion boundary |
|---|---|---|---|
| Technical feasibility | System logs, error cases, usability test | Concept diagram only | Can claim feasibility, not educational value |
| Governance efficiency | Process time, decision latency, workflow completion, staff interviews | Model accuracy | Can claim process improvement if measured in real workflow |
| Learning improvement | Learning outcomes, performance tasks, longitudinal traces | Satisfaction only | Avoid causal claim without comparison design |
| Teacher acceptance | Survey, interview, usage trajectory, workload data | Login counts only | Discuss acceptance cautiously |
| Organizational change | Policy/rule changes, routines, capacity building, longitudinal case data | One-time demo | Avoid claiming institutional transformation |
| Equity/fairness | Subgroup analysis, bias audit, accessibility evidence | Overall average | Report uneven effects and mitigation |
| Ethical safety | Consent, anonymization, human review, audit trail | “We value privacy” statement | Safety is a design claim only if safeguards are operational |

## Red-line rules

- Do not use model accuracy to prove education impact.
- Do not infer long-term change from one-time satisfaction data.
- Do not claim causality from cross-sectional survey alone.
- Do not claim fairness from aggregate results alone.
