# Technology-Organization Dual Topology

A governance architecture should show both technical layers and organizational accountability.

## Five technical-governance layers

1. **Infrastructure and data acquisition**: sensors, platforms, LMS, academic systems, student affairs systems, manual records.
2. **Data governance and interoperability**: standards, cleaning, authorization, anonymization, data quality and lifecycle.
3. **Intelligent analysis and Agent coordination**: analytics, LLM/Agent reasoning, alerting, recommendation and scenario simulation.
4. **Business application and decision support**: academic warning, teaching quality, resource allocation, student support, leadership dashboard.
5. **Human oversight and accountability**: review, interpretation, override, appeal, responsibility tracing and ethical review.

## Actor-responsibility mapping

| Actor | Responsibility |
|---|---|
| Education administrator | Policy alignment, resource coordination, accountability |
| School leader | Local rules, implementation pathway, organizational support |
| Teacher | Pedagogical interpretation, intervention, feedback |
| Student | Consent, data rights, feedback, self-regulation support |
| Technical team | Data security, model monitoring, system reliability |
| Expert/ethics committee | Risk review, fairness and privacy oversight |

## Anti-pattern

A diagram with only “data -> model -> output” is insufficient for education governance.
