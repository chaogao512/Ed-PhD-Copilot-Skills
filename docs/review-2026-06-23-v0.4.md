# Review: Ed-PhD-Copilot-Skills V0.4

## 评审结论

当前版本已从 V0.1 的“方向正确但较薄的技能集”提升到 **V0.4 样例验证版**。相较上一版，主要进步是：

- 7 个技能均已建立 `references/` 支撑体系。
- 形成 31 个 reference 文件，覆盖评分量表、致命缺陷、理论锚点、写作逻辑、方法细则、图示规则、审稿清单和 AI 辅助研究边界。
- 建立外部依据文件 `docs/evidence-base.md`，引入 UNESCO、European Commission、U.S. Department of Education、EU AI Act、设计科学研究、混合研究和 Delphi 方法等材料。
- 建立贯穿案例 `docs/case-multi-agent-smart-campus.md`。
- 生成 7 个技能样例输出，能够检验跨技能一致性。
- 建立结构检查脚本 `scripts/check_skill_structure.py`。

## 仍需改进的问题

### 1. 中文教育技术核心文献支撑不足

当前证据底座以国际组织、政策和通用方法论为主，尚未充分纳入《电化教育研究》《中国电化教育》《开放教育研究》等中文教育技术核心期刊中的真实论文模式。

**建议**：

- 建立 `docs/literature-map-edtech-governance.md`。
- 按主题整理 20-30 篇中文核心/SSCI 文献：智能教育治理、数字化转型、教师数字素养、人机协同评价、智慧校园、教育数据治理。
- 将文献模式转化为 reference 中的 worked examples 和 anti-patterns。

### 2. 方法论 reporting template 仍不够细

`mixed-methods-evidence-template` 已有方法清单，但还缺少可直接套用的报告模板。

**建议**：

- 为 Delphi 增加专家信息表、轮次记录表、共识阈值说明模板。
- 为问卷研究增加变量表、量表来源表、信效度报告模板。
- 为访谈研究增加访谈提纲、编码表和信度/可信度检查模板。
- 为 DSR 增加 build-evaluate-communicate 的设计记录模板。

### 3. 贯穿案例还只有“示范输出”，缺少失败回测

当前样例都是理想输出，尚未构造弱输入、错误输入或不完整输入来测试技能能否触发追问、降级或拒绝。

**建议**：

- 为每个技能增加一个 bad example。
- 测试技术先行、证据错配、伦理缺失、理论装饰化、政策口号化等失败输入。
- 把失败输出保存到 `docs/examples/failure-cases/`。

### 4. 结构检查脚本只检查存在性

当前脚本可以发现缺文件，但不能检查 reference 是否被 SKILL.md 引用，也不能检查 coverage matrix 是否与实际文件一致。

**建议**：

- 增加“reference 被引用检查”。
- 增加 coverage matrix 与实际文件一致性检查。
- 增加最小行数检查，避免空泛 reference。

### 5. 版本说明和发布节奏需要明确

README 已标注 V0.4，但缺少 changelog。

**建议**：

- 新建 `CHANGELOG.md`。
- 记录 V0.1、V0.4 的改造内容。
- 后续每轮增强都记录新增 references、样例和校验结果。

## 下一轮优先事项

1. 补中文教育技术核心文献地图。
2. 深化 `mixed-methods-evidence-template` 的方法报告模板。
3. 增加失败案例回测。
4. 强化结构检查脚本。
5. 新建 changelog。

## 当前状态判断

当前版本已经可以用于博士研究的结构化辅助，但还不应视为最终成熟版。它适合作为“教育信息化治理方向 Skill 框架 + 第一批方法规则 + 样例验证”的可用基础。要达到 V1.0，还需要真实文献模式和失败回测来提高稳定性。
