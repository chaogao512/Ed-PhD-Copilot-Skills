# Worked Example: Paper Skeleton

## Thesis statement

A multi-Agent smart campus governance mechanism can improve cross-department situational awareness and collaborative decision-making only when embedded in data governance rules, human oversight and organizational feedback loops.

## Theory-to-design matrix

| Theory | Design principle | Module | Evidence |
|---|---|---|---|
| Socio-technical systems | Jointly redesign technology and workflow | Five-layer architecture | Workflow comparison, interviews |
| Collaborative governance | Create shared decision routines | Cross-role governance dashboard | Meeting records, decision latency |
| Human-AI collaboration | Preserve human authority | Explanation and override node | Override logs, trust survey |
| Data governance | Control data quality and rights | Data standards and authorization | Data quality audit, access records |

## Chapter outline

1. Introduction: digital governance contradiction.
2. Theory and problem: socio-technical and collaborative governance lens.
3. Design principles: data interoperability, actor co-governance, human oversight, ethical safety.
4. Architecture: infrastructure, data governance, Agent coordination, application, oversight.
5. Validation: expert Delphi, pilot process logs, interviews.
6. Discussion: governance implications, limitations and implementation pathways.
