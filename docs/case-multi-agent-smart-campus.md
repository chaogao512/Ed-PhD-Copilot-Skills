# Case: 基于多 Agent 的高校智能校园治理机制研究

> 用途：作为 `Ed-PhD-Copilot-Skills` 的贯穿式测试案例。该案例不是最终论文，而是用于检验 7 个技能是否能在同一研究情境下保持概念、理论、方法和证据链一致。

## 研究背景

高校数字化转型推动教务、学工、教学质量、资源配置和学生支持等业务系统持续上线。但在实际运行中，许多学校仍面临数据孤岛、重复填报、平台堆叠、决策链条长、教师与学生反馈弱、智能推荐缺少解释与责任边界等问题。智能校园建设需要从“系统建设”转向“治理能力建设”。

## 核心问题

如何构建一种基于多 Agent 的高校智能校园治理机制，使不同业务部门、学校管理者、教师、学生和智能系统能够在数据治理规则、责任边界和人类监督机制下协同工作？

## 技术设想

- 建立多 Agent 协同机制，用于情境感知、任务分解、风险预警、证据汇聚和治理建议生成。
- 建立数据治理中台，统一数据标准、访问权限、脱敏规则和质量检查。
- 建立人类监督机制，要求关键建议必须由管理者、教师或专家复核后才能进入决策。
- 建立反馈回路，将治理行动、执行结果和 stakeholder 反馈纳入持续改进。

## 治理场景

1. 学业风险预警与支持。
2. 教学质量过程性评价。
3. 学生事务跨部门协同。
4. 教室与资源调度。
5. 学校领导决策支持。

## 利益相关者

| 角色 | 关注点 | 风险 |
|---|---|---|
| 学校管理者 | 治理效率、质量改进、责任可追踪 | 过度依赖系统建议 |
| 教师 | 教学反馈、学生支持、工作负担 | 数字倦怠、被动评价 |
| 学生 | 支持服务、公平与隐私 | 画像标签化、误判 |
| 技术团队 | 数据质量、系统稳定、安全 | 技术逻辑压倒教育逻辑 |
| 专家/伦理委员会 | 风险审查、价值对齐 | 审查流于形式 |

## 拟用理论

- 社会技术系统理论：解释技术系统与组织流程的共同优化。
- 协同治理理论：解释多主体参与、责任分担和协商决策。
- 人机协同/人在回路理论：解释智能建议、人工复核、责任追踪和反馈调控。
- 数据治理理论：解释数据标准、权限、质量、安全和生命周期。

## 拟用方法

- 设计科学研究：构建并迭代多 Agent 智能校园治理机制。
- 专家咨询或 Delphi：验证架构合理性、指标完整性和伦理安全性。
- 混合研究：结合系统日志、治理流程指标、教师/管理者问卷、访谈和案例观察。
- 文档分析：分析学校制度、数据管理规范和平台运行规则。

## 核心证据需求

| 研究主张 | 证据需求 |
|---|---|
| 提升治理流程效率 | 决策时长、任务流转记录、跨部门协同记录 |
| 改善教师和管理者支持体验 | 问卷、访谈、使用轨迹 |
| 保持人类主体性 | 解释查看记录、人工复核记录、覆盖/否决记录 |
| 数据治理可控 | 权限审计、脱敏规则、数据质量报告 |
| 伦理风险可控 | 知情同意、风险分级、申诉与人工复核机制 |

## 预期边界

本案例不能仅凭系统日志声称学习效果提升，不能仅凭满意度声称治理能力现代化，也不能在没有长期追踪的情况下声称组织转型完成。
