## Four-part logic chain

1. **政策背景**：高校教育数字化转型要求治理方式从经验驱动走向数据支撑、协同决策和持续改进。
2. **技术赋能**：多 Agent 和大模型技术能够支持情境感知、任务分解、跨部门协调和风险预警。
3. **现实异化**：现有智能校园建设常出现平台堆叠、数据孤岛、重复填报、算法黑箱和人类责任弱化。
4. **研究合法性**：本研究以社会技术系统和协同治理为理论基础，构建包含数据治理、人类监督和反馈改进的多 Agent 治理机制。

## Six-paragraph Introduction outline

1. **政策与时代需求**：说明教育数字化转型背景下，高校治理需要提升数据整合、协同决策和质量改进能力。
2. **技术潜能**：阐述多 Agent 系统在任务分解、智能分析、跨系统协同和治理建议生成方面的潜力。
3. **治理矛盾**：指出平台建设并不等于治理能力提升，数据孤岛、责任模糊和教师数字倦怠会削弱技术价值。
4. **理论桥接**：引入社会技术系统、协同治理和人在回路，提出研究问题：如何设计兼顾智能协同与人类责任的高校智能校园治理机制？
5. **本文方案**：提出五层架构：基础设施、数据治理、多 Agent 协同、业务应用、人类监督与问责，并说明混合研究验证路径。
6. **贡献**：贡献包括理论整合、治理架构、证据设计和实践路径四方面。

## V1.5 Evidence and Risk Boundary

- 文献表述若只是初步扫描，应写为“相关研究提示”，不得写成“系统综述表明”。
- 多 Agent 和大模型只能被表述为“可能支持协同治理的技术条件”，不能直接等同于治理能力提升。
- 若涉及学业预警、学生画像或教师评价，引言必须说明人类复核、解释申诉和风险监测是合法使用前提。
