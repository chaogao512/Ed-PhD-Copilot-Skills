# Ed-PdD-Copilot-Skills

面向教育技术领域、教育领导与管理专业、教育信息化治理方向的 AI Skills 项目。

本仓库计划以 `Supervisor-Skills` 的双轨结构为参考，但不直接复制其内容。后续改造将围绕教育信息化治理、智能校园治理、教育数字化转型、教育领导力、组织变革与教育研究方法，逐步沉淀适合博士研究、论文写作、课题设计和治理方案分析的 Copilot Skills。

## 项目定位

`Ed-PdD-Copilot-Skills` 不是通用科研写作提示词集合，而是面向教育技术与教育治理交叉研究的结构化技能库。它希望帮助研究者在以下场景中获得更稳定的 AI 辅助：

- 教育信息化治理方向选题判断与问题诊断
- 教育数字化转型、智能校园、数据治理、AI 治理等议题的研究设计
- 教育领导与管理专业论文的结构规划、理论嵌入与方法论校准
- 混合研究、设计科学研究、案例研究、政策文本分析等方法的执行检查
- 教育技术类 CSSCI、SSCI、核心期刊论文的投稿前自查
- 技术系统、组织结构、政策制度与多主体协同关系的图示设计

## 改造原则

本项目将参考 `Supervisor-Skills` 的组织方式，但核心内容需要面向教育学与治理研究重新建构。

1. 从技术性能导向转向教育治理问题导向。
2. 从算法改进叙事转向政策、组织、技术与人的协同叙事。
3. 从纯技术实验模板转向混合研究、设计科学研究和社科实证研究模板。
4. 从顶会审稿清单转向教育技术与教育管理期刊的理论、伦理、方法和政策合规清单。
5. 从神经网络图示逻辑转向多层治理架构、多主体协同与制度技术关系图示逻辑。

## 计划中的 Skills

首批技能建议从以下模块开始建设：

| 技能方向 | 初步用途 |
|---|---|
| `governance-idea-evaluator` | 评估教育信息化治理选题的政策价值、组织可行性、伦理风险和研究贡献 |
| `edtech-intro-drafter` | 生成教育技术/教育治理论文引言逻辑链，突出现实问题、理论缺口与技术赋能张力 |
| `mixed-methods-designer` | 辅助设计混合研究、设计科学研究、案例研究和政策文本分析方案 |
| `governance-framework-builder` | 构建智能校园、教育数据治理、AI 教育治理等研究框架 |
| `edtech-paper-reviewer` | 按教育技术与教育管理期刊标准进行投稿前自查 |
| `governance-figure-designer` | 设计多主体协同、治理架构、数据流与制度技术关系图 |

## 与 Supervisor-Skills 的关系

本仓库以 `Supervisor-Skills` 为基本模板来源之一，主要借鉴其“Handbook + Skills”的组织理念。实际内容将根据教育技术、教育领导与管理、教育信息化治理方向进行重写，不直接沿用原项目的计算机顶会、算法性能和纯技术论文评价框架。

如后续引用、改编或复用原项目中的具体文本、结构或资源，应保留原项目署名与许可证说明，并在对应文件中标注来源。

## 当前状态

项目处于初始化阶段。当前版本仅包含仓库定位、目录骨架和改造路线，尚未提供可直接安装使用的完整 Skill。
