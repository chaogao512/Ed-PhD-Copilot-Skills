# Allowed AI Uses in EdTech Research

| Task | Allowed AI role | Required verification |
|---|---|---|
| Literature organization | Summarize user-provided sources, build matrices | Check source existence and meaning |
| Theory comparison | Compare candidate theories | Verify against original literature |
| Outline generation | Produce alternative structures | User selects and revises logic |
| Method planning | Suggest evidence routes and checklists | Confirm feasibility and ethics |
| Figure design | Suggest layout and labels | Human checks disciplinary fit |
| Language polishing | Improve clarity | Ensure no new claims or citations |
| Code assistance | Clean, analyze or visualize data | Reproduce and inspect outputs |

## Disclosure-Aware Allowed Uses

| AI use | Usually acceptable when | Disclosure / record |
|---|---|---|
| Grammar, spelling and readability polishing | No new claim, citation, data or interpretation is introduced | Follow venue rules; often record internally |
| Outline alternatives | Human chooses structure and checks disciplinary logic | Record if it shaped manuscript organization |
| Literature matrix from user-provided PDFs | Sources are verified and summaries are checked against originals | Record tool, source set and verification method |
| Coding assistance for qualitative data | Data are de-identified or locally processed; researcher audits themes | Disclose if AI materially assisted coding or theme generation |
| Statistical/code assistance | Code is reproducible and outputs are inspected | Keep code, prompts or processing notes where feasible |
| Figure/layout drafting | Researcher verifies governance logic and labels | Disclose if generated figure content is materially AI-assisted |

Do not assume “language polishing” is automatically exempt from disclosure. Always check target journal, university and funder requirements.
