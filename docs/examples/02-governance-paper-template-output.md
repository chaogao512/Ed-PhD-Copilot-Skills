# Example Output: governance-paper-template

## 1. Paper type and thesis statement

论文类型：智能校园治理架构与人智协同机制论文。

核心论点：基于多 Agent 的智能校园治理机制只有在数据治理规则、人类监督、协同治理流程和伦理边界共同作用下，才能从技术平台升级为高校治理能力。

## 2. Theory-mechanism-system-evidence matrix

| Theory | Mechanism | System element | Evidence |
|---|---|---|---|
| 社会技术系统理论 | 技术与组织共同优化 | 五层智能校园治理架构 | 工作流变化、使用日志、访谈 |
| 协同治理理论 | 多主体共治与责任分担 | 跨部门决策仪表盘与任务流转 | 决策时长、协同记录、满意度 |
| 人机协同理论 | 人在回路、解释与否决 | 解释界面、人工复核、覆盖记录 | 复核日志、否决案例、信任问卷 |
| 数据治理理论 | 数据标准、权限与质量 | 数据治理中台 | 权限审计、数据质量报告 |

## 3. Section-by-section outline

1. 引言：提出高校智能校园建设中“平台扩张但治理能力不足”的矛盾。
2. 理论基础与问题情境：界定社会技术系统、协同治理、人机协同和数据治理。
3. 设计原则：提出数据可治理、主体可协同、建议可解释、责任可追踪、风险可规约。
4. 架构与机制：呈现基础设施、数据治理、Agent 协同、业务应用、人类监督五层架构。
5. 验证设计：采用专家咨询、流程日志、问卷访谈和案例观察。
6. 讨论：解释理论贡献、实践路径、伦理限制和推广条件。
7. 结论：总结机制价值，并限定其证据边界。

## 4. Practical pathway matrix

| Dimension | Pathway |
|---|---|
| 观念更新 | 从系统建设转向治理能力建设 |
| 机制重塑 | 建立跨部门数据与任务协同机制 |
| 评价升级 | 从结果评价转向过程证据和发展性支持 |
| 伦理规约 | 建立数据最小化、人工复核和申诉机制 |

## 5. Risks

最大风险是把多 Agent 写成技术说明书。必须让每个模块都对应治理问题、理论原则和证据指标。
