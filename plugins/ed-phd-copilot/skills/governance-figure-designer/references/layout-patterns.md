# Layout Patterns

| Pattern | Best for | Layout rule |
|---|---|---|
| Stacked layers | Smart campus architecture | Put infrastructure at bottom and human oversight/accountability at top or side rail |
| Concentric ecology | Macro-meso-micro governance | Put learner/teacher practice at center, institution and policy outside |
| Swimlane | Multi-actor workflow | One lane per actor; arrows show responsibility transfer |
| Feedback loop | Human-AI collaboration | Close the loop with evaluation and adjustment; mark override points |
| Matrix | Claim-evidence or actor-responsibility mapping | Rows are actors/claims, columns are responsibilities/evidence |
| Before-after contrast | Diagnosing digital alienation | Left: fragmented state; right: redesigned governance mechanism |
| Responsibility swimlane | High-risk workflow, AI assessment, data access | One lane per responsible actor; mark review, override, appeal and audit points |
| System-card side panel | AI model or Agent system description | Put intended use, non-use, limits, oversight and monitoring beside the main flow |
| Dataset-card side panel | Education data source description | Put data source, subject, consent, quality, retention and deletion beside the data flow |
| Risk register table | AI/data governance risk | Columns: risk, affected group, severity, safeguard, owner, monitoring evidence |

## V1.5 Layout Rules

- High-risk AI diagrams must include either a responsibility swimlane or a visible oversight side rail.
- Data-heavy diagrams must include dataset-card information; do not show “all data” as a neutral input.
- AI-system diagrams must include model/system-card information; do not show the model as an unbounded black box.
- Evidence-chain diagrams must separate literature scan, systematic review, empirical data and expert consultation.
